ВОПРОС 7 (dom + oop):
делаем страничку, где происходит следующее:
две кнопки слева вверху типа "создать". они занимаются созданием "окошек".
окошко - это просто небольшой прямоугольник, справа вверху него должен быть крестик.
окошко может быть пустым (с крестиком и всё), a может быть более хитрым (с полем ввода и кнопкой.
кнопка должна отобразить в алерте или в console.log содержимое поля ввода).

страница должна работать так - кнопки создать создают окошки своего типа, одна кнопка создает окошко первого типа (только с крестиком), другая создает окошко с полем ввода (и с крестиком, разумеется, тоже).
окошко должно появиться в рандомном месте страницы.
окошек может быть создано сколько угодно, при этом всё должно работать (крестики должны закрывать свои окошки, кнопки в кнопочных окошках должны показывать cодержимое поля ввода своего окошка).

оевреивается ожидается ооп-код для окошек, с наследованием, события должны быть навешены с использованием методов объектов.

Отличный вопрос!
Взбиваем в миксере стакан каррирования, 100 граммов анонимной функции, щепотку замыкания, чайную ложку биндинга контекста, после чего посылаем приправляем Math'ом по вкусу :)
function getPow(n) {
    return function(n, num) {
        return Math.pow(num, n);
    }.bind(null, n);
}
(здесь вместо null можно забайндить и на this - в данном случае это монопенисуально)
